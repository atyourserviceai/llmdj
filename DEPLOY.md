# LLMDJ Demo Deployment Guide

This document covers deployment of the LLMDJ (AI Music Discovery Agent) demo, which demonstrates OAuth integration with both the atyourservice.ai platform and Spotify API.

## Service Overview

LLMDJ is an AI-powered music discovery agent that:

- Authenticates users through atyourservice.ai OAuth
- Integrates with Spotify Web API for music playback control
- Uses the Gateway for AI request routing
- Provides user-specific agent instances with persistent state
- Demonstrates dual OAuth provider integration

## Prerequisites

- Website service deployed and operational (provides OAuth)
- Gateway service deployed and operational (provides AI routing)
- Spotify Developer App configured with appropriate permissions
- Cloudflare Workers account with appropriate permissions

## Configuration

### Environment Variables

#### Required for All Environments

```bash
# Environment identification
SETTINGS_ENVIRONMENT="local|staging|production"

# Service URLs (environment-specific)
GATEWAY_BASE_URL="<gateway-url>"
OAUTH_PROVIDER_BASE_URL="<website-url>"

# Spotify integration
SPOTIFY_CLIENT_ID="<spotify-client-id>"
SPOTIFY_REDIRECT_URI="<llmdj-url>/spotify/callback"
```

#### Environment-Specific URLs

| Environment    | Gateway URL                                | OAuth Provider URL                      | LLMDJ URL                                         |
| -------------- | ------------------------------------------ | --------------------------------------- | ------------------------------------------------- |
| **Local**      | `http://localhost:48787`                   | `http://localhost:5173`                 | `http://localhost:5174`                           |
| **Staging**    | `https://staging.gateway.atyourservice.ai` | `https://dev.atyourserviceai.pages.dev` | `https://llmdj-staging.augmentedmind.workers.dev` |
| **Production** | `https://gateway.atyourservice.ai`         | `https://atyourservice.ai`              | `https://llmdj.com` (custom domain)               |

### Spotify Developer App Setup

#### Required Spotify App Settings

- **App Name**: LLMDJ (or appropriate environment name)
- **App Description**: AI Music Discovery Agent
- **Redirect URIs**:
  - Staging: `https://llmdj-staging.augmentedmind.workers.dev/spotify/callback`
  - Production: `https://llmdj.augmentedmind.workers.dev/spotify/callback`
- **Scopes Required**:
  - `user-read-playback-state`
  - `user-modify-playback-state`
  - `user-read-currently-playing`
  - `playlist-modify-public`
  - `playlist-modify-private`

## Troubleshooting

### Common Issues

#### OAuth Flow Failures

- **Symptoms**: Authentication redirects fail or loop
- **Check**: OAuth client configuration in website database
- **Fix**: Verify redirect_uris match exactly, including protocol and port

#### Spotify Integration Errors

- **Symptoms**: Spotify features don't work
- **Check**: Spotify app configuration and redirect URIs
- **Fix**: Verify Spotify app settings and required scopes

#### Gateway Communication Errors

- **Symptoms**: AI requests fail
- **Check**: Gateway URL configuration and authentication
- **Fix**: Verify GATEWAY_BASE_URL and token passing

#### Agent State Issues

- **Symptoms**: Agent doesn't remember user context
- **Check**: Durable Object configuration and user isolation
- **Fix**: Verify agent routing by user ID

## Security Considerations

### Token Management

- JWT tokens are never logged in production
- Spotify tokens are stored securely in agent state
- All external API communications use HTTPS

### User Data Protection

- User Spotify data is isolated per agent instance
- No cross-user data leakage
- Spotify permissions are minimal required scope

### OAuth Security

- State parameter validation in OAuth flows
- CSRF protection on authentication endpoints
- Secure redirect URI validation
